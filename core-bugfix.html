A Note About Load Order and Conflict Resolution

LOOT is not used in this guide. The best way to sort your load order is to use FNVEdit to view conflicts and manually sort your load order based off that and then create a custom conflict resolution patch. The guide will provide a fully-modular handmade conflict resolution FOMOD (if necessary) along with the proper load order for each module, so you do not need to worry about either of those if you are just following the guide. This means that any mods listed can be left out without missing out on the pre-made conflict resolution/load order. However, if you plan on adding any big mods, you will need to familiarize yourself with manually solving conflicts in xEdit. I suggest this tutorial for familiarizing yourself with the tool and for making custom conflict resolution.

A Note About the Plugin Limit

Fallout New Vegas has a safe plugin cap of about ~120 (no one knows the exact number) in vanilla, but the Mod Limit Fix in Core - Utilities raises that to 255. The guide will stay far below 255, at ~140 plugins with the full guide, including optionals. If you somehow add enough of your own mods to get near the plugin cap, it is not recommended to use automated merging tools like Merge Plugins Standalone or zMerge. If you need to merge, manually merging with FNVEdit is the best and safest way to do it.

Creating a Separator in Mod Organizer 2

  • Right-click in the left pane of MO2 and select Create Separator
  • Input Core into the pop-up and select OK
  • Create another separator below that called Bug Fixes + QOL
